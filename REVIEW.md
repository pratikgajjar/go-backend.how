## Risks & known caveats

- **fdyno still draft.** Build with `--buildDrafts` if you want to
  preview locally; `hugo --environment production` (production deploy
  command) excludes it correctly.
- **Theme/data coupling.** `data/themes.yaml` and CSS variable
  references in `main.css` must stay in sync. If a future palette is
  added in only one place, theme will fall back to `vanilla`.
- **Heading bump.** The +1 bump in `render-heading.html` is scoped to
  `Section=posts AND Kind=page`, but assumes all `posts/` content is
  article-shaped (single article-title h1 wrapping markdown content).
- **Font subset is content-driven.** New characters require font regen.
  Detector flags this indirectly via `js_console_error_or_404` (browser
  fallback chain) but not directly.
- **`_headers` is Cloudflare-Pages-specific.** Other hosts use different
  syntax (`netlify.toml`, nginx config, etc.).
- **External URL rot ongoing.** Phase 7 fixed 4 broken/redirected URLs.
  Re-run `scripts/site-quality-check.py --check-external` periodically;
  cache invalidates after a week.
- **Long titles.** `valkey-part-1` is 69 chars without suffix —
  author's choice (post title alone exceeds 60). Detector flags
  `title_too_long > 75` only.
- **autoresearch.{md,jsonl,ideas.md} + REVIEW.md.** Agent meta files.
  Either keep in repo as a research diary, or strip before push if
  intended to be private. They are not deployed (not in
  `content/`).
